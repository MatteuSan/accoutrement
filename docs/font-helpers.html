Internal WebFont Helpers

These internal tools are not safe for public use – but we’ve documented the logic for transparency, and to encourage contributions.

  'italic',
  'oblique',

A constant list of valid font-style settings (not including normal, which is also a valid weight).

@function _font-get() [private]

Get a font from the $fonts map, based on font access-key. This is similar to the Sass map-get() function, but the map is always your global $fonts map, and we allow internal alias references - so that multiple keys can be used for a single font.

Parameters & Return

$key:

The font access-key defined in your $fonts map, or an alias key that redirects to the appropriate font.

@return (map)

Map of all the font-data associated with the given key.

@error

#{$key} is not defined in your $fonts configuration map.

Requires

$fonts (Map)

Used By

@function font-family()

@mixin font-face()

@function _font-url() [private]

Interpolate a font path and format-type to create the proper font-src import URL. This private function does not prepend any global $font-path, leaving that to the public font-url() function.

Since 4.0.0:

Added to format font-paths correctly, with file extension and suffix.

Parameters & Return

$path: (string)

The base path for this font and format. If no file extension is included, we add one based on the format desired. Local and base64 font names & URI’s are returned without any change.

$format: (string)

Short-form of the format to import, e.g. woff or eot.

$svgid: (string)

SVG fragment-identifier to append on svg fonts. The initial # character is optional, and will be added when missing.

$iefix: (boolean)

Optionally append an ?#iefix suffix to eot fonts.

@return (string)

Font-format URL for importing.

Used By

@function _font-src() [private]

@function _normalize-variant-data() [private]

@mixin _import-font-face() [private]

@function _font-src() [private]

Get the font src value for a given font path and format, including both url() and format(). This also sorts formats, organizing them into the proper output order for browser-compatability.

Since 4.0.0:

No longer accepts the $format parameter.

Parameters & Return

$path: (string | map)

Path to the font files (leaving off the file format indicator), or data-uri for font-src.

$format: (string)

Short-form (e.g. ‘eot’ or ‘otf’) font-format to import.

$suffix: null (string)

SVG or EOT-iefix suffix to be added to font url.

@return (list)

CSS-ready url() format() combination for font-face src output.

@function _normalize-variant-data() [private]

Convert variant values into a consistent format - to allow shorthand APIs, while providing consistent data formats to the internal functions. Strings are converted to maps, and format-paths are interpolated to create a self-contained map of variant data.

Since 4.0.0:

Added to normalize string and map variant syntaxes into a single (map) format for internal use.

Parameters & Return

$data: (string | map)

A string path for interpolation, or a map of explicit format/path pairs for one font-variant.

$formats: () (list)

A list of formats to interpolate with generic paths, to create format/path values. If a path is provided without formats, we’ll return format-error - allowing the function-call origin to provide a more detailed error message.

@return (map)

A map of formats and urls for the given font-variant, merging interpolated and explicit format/paths into a single map.

Requires

@function _font-url() [private]

$_font-format-options [private]

Used By

@mixin _import-font-face() [private]

@mixin _import-font-face() [private]

Compile provided font-variant data into a CSS @font-face import.

Since 4.0.0:

  • Changed parameters to handle new syntax for variant-data and font-formats.
    • Errors when path is provided without formats.
    • Outputs updated font-face syntax, based on fontsquirrel approach.

Parameters & Output & Return

$name: (string)

The font name.

$variant: (string | list)

The font-style and font-weight properties, e.g. bold or ('bold', 'italic').

$data: (string | map)

Path to the font files (leaving off the file-format extension), or map of format/path pairs.

$formats: () (list)

Optional list of font formats (e.g. ‘ttf’, ‘otf’, ‘woff’) to import, in the order given. If formats are not explicitly given here, we fallback to formats in a data-map.

$<key>.unicode-range: (list | null)

Optionally import only specific unicode ranges. See MDN unicode-range documentation for details.

{CSS output} (code block)

@font-face CSS-block for importing a given font variant.

@error

(#{$name}) Font-path interpolation

Examples

scss string $data 
@include _import-font-face(
  $name: 'Franklin Gothic',
  $variant: ('bold' 'italic'),
  $data: 'path/to/franklin-bolditalic-font',
  $formats: ('woff2' 'woff')
);
scss map $data 
@include _import-font-face(
  $name: 'Franklin Gothic',
  $variant: 'normal',
  $data: (
    'local': 'Franklin Gothic' ('franklin-gothic'),
    'woff2': 'path/to/franklin-bolditalic-font.woff2',
    'ttf': 'path/to/franklin-bolditalic-font.ttf',
  )
);
